# 校园外卖点餐管理系统项目说明

## 一、项目背景

校园外卖点餐管理系统面向校内学生和后台管理员。学生可以浏览商家与菜品、加入购物车、填写收货地址并提交订单；管理员可以维护商家、菜品和订单状态。

## 二、角色设计

系统包含两类用户：

- 学生用户：注册登录、浏览菜品、购物车、提交订单、查看订单。
- 管理员：登录后台、管理商家、管理菜品、管理订单。

## 三、数据库设计

系统共设计 9 张数据表，满足实训要求中“至少 6 张表”的条件。

| 表名 | 中文名 | 说明 |
| --- | --- | --- |
| users | 用户表 | 保存学生和管理员账号 |
| merchants | 商家表 | 保存校内外卖商家信息 |
| categories | 菜品分类表 | 保存主食、饮品、小吃等分类 |
| dishes | 菜品表 | 保存菜品名称、价格、库存等 |
| addresses | 收货地址表 | 保存学生宿舍地址 |
| orders | 订单表 | 保存订单主信息 |
| order_items | 订单明细表 | 保存每个订单包含的菜品 |
| reviews | 评价表 | 保存用户评价，作为扩展功能 |
| announcements | 公告表 | 保存系统公告 |

## 四、主要功能模块

### 1. 用户模块

- 学生注册
- 用户登录
- 用户退出
- 根据角色跳转学生端或管理员端

### 2. 菜品浏览模块

- 展示商家列表
- 展示菜品列表
- 支持按菜品、商家、分类搜索

### 3. 购物车模块

- 添加菜品到购物车
- 修改菜品数量
- 删除购物车菜品
- 自动计算总价

### 4. 订单模块

- 选择收货地址
- 提交订单
- 生成订单编号
- 查看我的订单

### 5. 后台管理模块

- 后台数据概览
- 商家新增、编辑、删除
- 菜品新增、编辑、删除
- 订单状态修改

## 五、系统结构

项目采用传统 Java Web 三层结构：

- JSP：页面展示
- Servlet：请求控制
- DAO：数据库访问
- Model：实体类
- MySQL：数据存储
